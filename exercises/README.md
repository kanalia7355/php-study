## 課題の取り組み方

### ステップ1：課題に挑戦
1. 各章の課題集を開く
2. 課題の要件を確認
3. ヒントを参考にしながら実装
4. 拡張チャレンジに挑戦（オプション）

### ステップ2：解答と比較
1. 解答集を開く
2. 自分の解答と比較
3. 違いがあれば理解を深める
4. ベストプラクティスを学ぶ

### ステップ3：実践
1. 解答のコードを実際に実行
2. 動作を確認
3. コードを改造して実験

## 評価基準

全ての課題は以下の観点で評価されます：

- **機能**: 要求された全ての機能が正しく動作する
- **コード品質**: 可読性が高く、適切にコメントされている
- **エラー処理**: 例外ケースに適切に対応している
- **パフォーマンス**: 大きなデータでも適切に動作する
- **再利用性**: 汎用的に使える設計になっている
- **オブジェクト指向**: 適切なクラス設計とカプセル化
